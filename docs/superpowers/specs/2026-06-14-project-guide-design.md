# 项目熟悉指南设计

## 目标

为首次接手并准备继续开发本项目的维护者编写一份综合指南。文档先帮助读者运行和体验系统，再沿真实业务流程深入前端、后端、多智能体、数据库和 RAG 源码。

## 交付物

在仓库根目录新增 `PROJECT_GUIDE.md`，使用中文编写。

## 内容结构

1. 项目定位、主要功能和技术栈
2. 五分钟快速启动与依赖准备
3. 环境变量和数据库用途
4. 目录结构和推荐阅读顺序
5. 登录、会话、聊天、文档上传的数据流
6. 前端入口、路由、页面和状态管理
7. FastAPI 入口、认证和业务路由
8. LangGraph 多智能体、工具和记忆机制
9. RAG 文档提取与检索
10. API 速查表
11. 测试、调试和常用命令
12. 当前实现中的不一致、风险和后续切入点

## 编写原则

- 以当前源码为事实来源，不把现有 README 当作唯一依据。
- 不展示 `.env` 中的密钥、密码或完整数据库连接串。
- 明确区分“已经注册并可用的功能”和“代码存在但尚未接入主应用的功能”。
- 使用具体文件路径帮助读者继续阅读源码。
- 启动命令以 Windows PowerShell 和当前目录结构为准。
- 不修改业务代码，不承诺源码中尚未实现的行为。

## 验收标准

- 新开发者能根据文档启动前后端并找到 API 文档。
- 能说明一次聊天请求经过的主要模块。
- 能区分 MySQL、PostgreSQL 会话库、长短期记忆库的职责。
- 能找到前端页面、后端路由、多智能体和 RAG 的入口文件。
- 文档列出的路由、命令和文件路径均与当前仓库一致。
- 文档明确记录当前可见的接口接线和工程维护风险。
